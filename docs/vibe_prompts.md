## 🚀 ПРОМПТ №1 — ОСНОВА ПРИЛОЖЕНИЯ И ХРАНИЛИЩЕ

```
Создай одностраничное веб-приложение "Финансовый трекер" на чистом HTML/CSS/JavaScript
(без фреймворков, без серверной части).

ХРАНЕНИЕ ДАННЫХ:
- Все расходы хранятся в localStorage под ключом "expenses"
- Каждый расход — объект: { id, date, amount, category_id, description, regret_score, created_at }
- При первом запуске загружай конфигурацию бюджета из budget_config.json (fetch)
- Конфиг тоже кешируй в localStorage под ключом "budget_config"

СТРУКТУРА CSV ДЛЯ ЭКСПОРТА:
Дата,Сумма,Категория,Описание,Оценка_ненужности,Создано
2026-04-01,360,food_out,Boltay — кофе и круассан,2,2026-04-01T09:15:00

ПОЛЯ:
- date: YYYY-MM-DD
- amount: число в рублях
- category_id: id из конфига
- description: текст свободной формы
- regret_score: число 0-5 (0 = нужная покупка, 5 = зря потратил)
- created_at: ISO timestamp

ДИЗАЙН — тёмная тема, финансовое приложение, шрифт Geist Mono или JetBrains Mono для цифр,
Manrope для текста. Цветовая схема: фон #0A0A0F, акцент #00FF88 (зелёный неон),
предупреждения #FF4444, нейтральный #888. Карточки с border-radius 16px,
тонкие border rgba(255,255,255,0.08). Ощущение Bloomberg Terminal + modern fintech.

Начни с создания app.js с полным слоем данных:
- saveExpense(expense)
- getExpenses(dateFrom, dateTo)
- deleteExpense(id)
- getMonthlyStats(year, month) → { byCategory, totalSpent, totalBudget }
- exportToCSV(dateFrom, dateTo) → скачивает файл
- loadBudgetConfig() → возвращает конфиг из localStorage или fetch
```

## 🎨 ПРОМПТ №2 — ДАШБОРД (index.html)

```
Создай дашборд index.html для финансового трекера.

ПОДКЛЮЧИ: app.js, budget_config.json

БЛОКИ НА СТРАНИЦЕ:

1. ШАПКА
   - Текущий месяц и год
   - Большое число: потрачено сегодня в рублях
   - Маленький текст под ним: "из ~XXXX ожидаемых сегодня"
     (ожидаемые на сегодня = total_variable_budget / кол-во дней в месяце * номер сегодняшнего дня)
   - Кнопка "+ Добавить расход" (ведёт на add.html)

2. ПРОГРЕСС МЕСЯЦА
   - Горизонтальная полоса: прошло X из Y дней
   - Потрачено X ₽ из Y ₽ бюджета переменных расходов
   - Если потрачено > 80% бюджета и месяц ещё не кончился — красная пульсирующая точка рядом

3. КАРТОЧКИ ПО КАТЕГОРИЯМ (сетка 2 колонки на мобиле, 3-4 на десктопе)
   Каждая карточка:
   - Эмодзи + название категории
   - Прогресс-бар (потрачено / бюджет)
   - "XXXX ₽ из XXXX ₽"
   - Если > 100% — карточка светится красным, текст "ПРЕВЫШЕНО на X ₽"
   - Если > 80% — карточка желтоватая, текст "Осталось X ₽"
   - Если < 50% — зелёный акцент

4. ПОСЛЕДНИЕ 5 РАСХОДОВ
   Список: дата · категория · описание · сумма · regret_score (звёздочки или точки)

5. МИНИ-ИНСАЙТ (генерируется локально, без AI)
   Примеры логики:
   - Если food_out > 60% бюджета за первые 2 недели → "🔥 Еда вне дома: уже 60% бюджета за полмесяца"
   - Если сегодня потрачено > дневной нормы × 1.5 → "📊 Сегодня выше нормы в 1.5 раза"
   - Если 3 дня подряд regret_score >= 4 → "🤔 Последние 3 дня — сомнительные покупки"

АНИМАЦИИ: плавное появление карточек с stagger (animation-delay), счётчики цифр анимируются
при загрузке страницы (число нарастает от 0 до значения за 600мс).

АДАПТИВНОСТЬ: работает на мобиле (основной сценарий), планшете, десктопе.
```

## ➕ ПРОМПТ №3 — ФОРМА ДОБАВЛЕНИЯ РАСХОДА (add.html)

```
Создай страницу add.html — добавление нового расхода.

ФОРМА:

1. СУММА
   - Большое поле ввода по центру, шрифт моно, размер 48px
   - Автофокус при открытии
   - Клавиатура: только цифры (inputmode="numeric")
   - Показывай сумму сразу форматированной: "1 234 ₽"

2. КАТЕГОРИЯ
   - Горизонтальный скролл карточек-пилюль с эмодзи
   - Категории из budget_config.json
   - При выборе показывай: "Осталось X ₽ в этой категории в этом месяце"
   - Если категория уже превышена — показывай красный бейдж "ЛИМИТ"
   - Категории из budget_config.json (вставь все 10 категорий из файла)

3. ОПИСАНИЕ
   - Текстовое поле, placeholder: "На что потратил? (Boltay, кофе и круассан)"
   - Необязательное

4. ОЦЕНКА НЕНУЖНОСТИ (regret_score)
   Заголовок: "Насколько эта покупка была нужна?"
   5 кнопок в ряд:
   0 — "Необходимо" (зелёный)
   1 — "Нужно" (светло-зелёный)
   2 — "Окей" (жёлтый)
   3 — "Сомнительно" (оранжевый)
   4 — "Лишнее" (красный)
   5 — "Зря" (тёмно-красный)

5. ДАТА
   - По умолчанию сегодня
   - Кнопки: "Сегодня", "Вчера", или выбор даты

6. КНОПКА "СОХРАНИТЬ"
   - После сохранения — анимация успеха (зелёная вспышка), потом редирект на дашборд
   - Или кнопка "Добавить ещё" для быстрого ввода нескольких расходов подряд

UX-детали:
- При regret_score >= 4 показывай сообщение: "Точно нужно? Подожди 24 часа перед дорогими покупками"
- Если сумма > 5000 — автоматически ставить фокус на поле описания
- Haptic feedback на мобиле (navigator.vibrate(10)) при выборе категории
```

## 📊 ПРОМПТ №4 — ИСТОРИЯ И ЭКСПОРТ (history.html)

```
Создай страницу history.html — история расходов и экспорт.

ФИЛЬТРЫ (сверху, горизонтальный ряд):
- Период: эта неделя / этот месяц / прошлый месяц / произвольный (date picker)
- Категория: все или конкретная (дропдаун с эмодзи)
- Сортировка: по дате ↓ / по сумме ↓ / по regret_score ↓

СПИСОК РАСХОДОВ:
Каждая строка:
- Левая часть: дата (ДД.ММ) + описание + категория (эмодзи + название маленьким текстом)
- Правая часть: сумма жирно + regret_score точками (● = потраченный балл, ○ = нет)
- Свайп влево на мобиле = удалить (с подтверждением)
- Группировка по дням с заголовком-датой и итогом за день

СУММАРНАЯ СТАТИСТИКА ЗА ПЕРИОД (карточка сверху):
- Потрачено всего
- Средний расход в день
- Самая дорогая категория
- Средний regret_score (если > 2.5 — красным)
- Кол-во покупок с regret >= 4 и их суммарная стоимость ("Сомнительных трат: X на сумму Y ₽")

ЭКСПОРТ:
Кнопка "Экспортировать CSV" — скачивает расходы за выбранный период.
Формат CSV:
Дата,Сумма,Категория,Описание,Оценка_ненужности
2026-04-01,360,Еда вне дома,Boltay — кофе и круассан,2

Подсказка под кнопкой: "Скинь этот файл Claude для анализа трат"
```

4. ОПАСНАЯ ЗОНА
   Кнопка "Очистить все расходы этого месяца" (с подтверждением)
   Кнопка "Сбросить всё" (с двойным подтверждением)

Все изменения сохраняются в localStorage автоматически при blur из поля.

Мой бюджет по категориям:
- Продукты: 25 000 ₽ (потрачено: X ₽)
- Еда вне дома: 8 000 ₽ (потрачено: X ₽)

Мой доход: 295 000 ₽. Фиксированные расходы: 84 856 ₽.

1. Где я вышел за бюджет и почему (на основе описаний)?

3. 3 конкретных изменения на следующий месяц
4. Сколько я мог бы откладывать при оптимизации
---

3. Копируем этот промпт в буфер обмена

Дополнительно — кнопка "Скачать промпт.txt" сохраняет этот текст как файл.
